> # Docker
> ----------------------------------------------------
> ### 1. Docker란
>    
>  * 정의 : 애플리케이션을 컨테이너라는 표준 단위로 패키징해 빌드, 배포, 실행, 업데이트, 관리를 빠르게 해주는 오픈 소스 플랫폼
>  * 핵심 : 컨테이너 안에 앱 실행에 필요한 것(코드, 런타임, 라이브러리, 시스템 도구)을 모두 포함해 환경 차이로 인한 오류를 원천 차단
>   
> ### 2. 컨테이너 vs 가상머신 다른 점
>    
> | 구분 | 컨테이너(Docker) | 가상머신(VM)|
> |------|------|------|
> | 구조 | 호스트 커널 공유, 프로세스 단위 격리 | 하드웨어 가상화 + 게스트 OS 통째로 실행 |
> | 기동 속도 | 매우 빠름(보통 초 단위) | 느림( 수십 초~수분) |
> | 자원 사용 | 가벼움(동일 서버에 더 많은 인스턴스) | 무거움(CPU/메모리 더 필요) |
> | 격리 | 충분히 강력(네임스페이스, cgroups) | 강력(완전한 OS분리) |
> | 사용 예 | 마이크로서비스, CI/CD, 대규모 배포 | 서로 다른 OS가 꼭 필요할 때 |
>
> ### 3. Docker의 작동방식
>    
>  * 리눅스 네임스페이스 : 프로세스/네크워크/파일시스템을 논리적으로 분리
>  * cgroups : CPU. 메모리 등 자원 사용 한도를 제어
>  * 이미지 : 컨테이너를 만드는 불변 템플릿(버전.레이어 캐시 가능)
>  * 컨테이너 : 이미지를 실제로 실행한 인스턴스
>      
>      ##### os를 통째로 켜지 않으니 더 가볍고, 더 빠르고, 더 많이 돌릴 수 있다.
>
> ### 4. 주요 사용 사례
>    
>  * 마이크로서비스 : 서비스별로 컨테이너를 쪼개 독립 배포/확장
>  * CI/CD자동화 : 빌드. 테스트. 배포 파이프라인을 표준 컨테이너로 속도 극대화
>  * 데이터 처리/MLOps : 라이브러리 충돌 없이 일관 환경으로 재현성 보장
>  * CaaS & 클라우드 배포 : AWS Fargate 등과 연계해 운영 부담 최소화
>
> ### 5. Docker의 장점
> 
>   * 이식성 & 환경 일관성 
>     * 컨테이너 한 번 만들면 Docker만 있으면 어디서나 동일 동작, 배포 이슈 급감
>    
>   * 속도 & 생산성
>     *  커널 공유 기반 경량화로 기동/빌드/배포가 빠름. 실험->릴리즈 주기가 짧아짐
>    
>   * 효율 & 비용 최적화 
>     *  VM 대비 자원 소모가 적어 같은 서버에 더 많은 워크로드. 인프라 비용 절감
>    
>   * 격리 & 유연성
>     *  서비스 간 충돌 방지, 장애 전파 최소화, 마이크로서비스 아키텍처 구현에 필수

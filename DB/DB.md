> # DB 설계기초
> --------------------------------------------------------------------------------------
> 
> ### 1. 개념 설계(무엇을 저장할지)
>
>  * 명사(엔티티) 추출
>
> ### 2. 논리 설계(어떻게 나눌지)
> 
>    현실의 데이터를 어떤 테이블로 나누고, 그 테이블들이 어떤 관계를 맺는지 결정하는 단계
> * 오직 구조와 관계만 고민
>
> 
> *  #### 논리 설계 검증법
>       1. 한개의 이벤트만 단독으로 의미가 있는가?
>       2. 특정 데이터만 삭제 가능한가?
>       3. 통계 쿼리를 바로 짤 수 있는가?
>       4. 확장(영상, 음성)이 쉬운가?
>  
> ### 3. 물리 설계(어떤 타입/제약을 쓸지)
> *  PK/FK
> *  BOOLEAN
> *  VARCHAR
> *  TIMESTAMP

> ### 4. DB 설계의 핵심 규칙
> 1. 한 테이블 = 한 책임
> 2. 중복 데이터 저장 금지
> 3. 시간은 반드시 저장
> 4. 파일은 DB에 저장하지 않음.
> 5. "나중에 질문할 수 있는 구조"로 설계
